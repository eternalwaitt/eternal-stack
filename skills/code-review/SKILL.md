# Code Review

Lead with findings:

1. Check correctness, safety, verification, rollout, and maintenance risk.
2. Separate proven issues from assumptions.
3. Point to exact files, lines, commands, or plan sections.
4. Recommend the smallest fix that closes the risk.
5. Say clearly when no blocking findings remain.
