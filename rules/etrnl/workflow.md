# Workflow

- Use hooks for enforcement, skills for repeatable procedures, and startup files only for routing.
- Use `etrnl-brainstorm` when requirements are still fuzzy.
- Use `etrnl-plan` for multi-step implementation plans; write to disk, review, improve, and finalize.
- Use `etrnl-execute` only when the user explicitly asks to execute a written plan.
- Use `etrnl-review` or `etrnl-stress-test` for gap reviews, risks, and final passes.
- Use `etrnl-code-health` for whole-codebase health, no-skips audits, dead code, repo rot, docs health, and PR gates.
- Keep chat summaries short when the durable artifact is a file.
